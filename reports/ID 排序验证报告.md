## 🔧 修正内容

### 原排序逻辑

```python
issues = query.order_by(QualityIssue.created_at.desc())
```

**问题**: 批量导入时所有记录的 `created_at` 时间相同，排序不稳定

### 修正后排序逻辑

```python
# 使用 ID 降序排序（ID 越大越新），确保排序稳定
issues = query.order_by(QualityIssue.id.desc())
```

**优势**: 
- ✅ ID 是自增主键，严格递增
- ✅ ID 越大 = 记录越新
- ✅ ID 越小 = 记录越旧
- ✅ 排序结果稳定可靠

## 📄 排序规则说明

### 为什么使用 ID 排序而不是时间排序？

1. **时间字段可能相同**
   - 批量导入时，所有记录的 `created_at` 可能相同
   - 导致排序结果不稳定

2. **ID 严格递增**
   - ID 是 AUTO_INCREMENT 主键
   - 每条记录的 ID 都唯一且递增
   - 排序结果稳定可靠

3. **ID 与时间正相关**
   - ID 越大 = 插入越晚 = 记录越新
   - ID 越小 = 插入越早 = 记录越旧

### 排序规则

```sql
-- 新到旧排序（最新记录在前）
ORDER BY id DESC

-- 旧到新排序（最旧记录在前）
ORDER BY id ASC
```
