## 1. 项目概述

### 1.1 项目背景

为服装公司建立 QC 质量管理系统，用于管理售后质量问题记录、质检人员考核、工厂质量评估。

### 1.2 项目目标

- ✅ 实现质量问题快速录入（移动端）
- ✅ 自动记录提交人 QC 信息
- ✅ 支持波次号/货品编码自动关联
- ✅ 高并发数据处理能力（75+ QPS）
- ✅ 数据完整性保证（99.9%+ 成功率）

### 1.3 用户角色

| 角色 | 权限 | 说明 |
|------|------|------|
| **admin** | 全部权限 | 系统管理员 |
| **manager** | 管理权限 | 质检经理 |
| **qc** | 录入权限 | 质检员 |
| **viewer** | 只读权限 | 查看人员 |

### 1.4 使用场景

1. **QC 质检员**: 现场录入质量问题
2. **质检经理**: 查看统计数据、考核 QC 绩效
3. **系统管理员**: 用户管理、系统维护
4. **管理层**: 查看质量报告、工厂评估

## 2. 功能需求

### 2.1 用户认证

**功能描述**: JWT Token 认证

**接口**:
- `POST /token` - 用户登录
- `GET /api/user/me` - 获取当前用户信息

**要求**:
- Token 有效期：1 天
- 密码加密：SHA256
- 支持角色权限控制

### 2.2 问题录入

**功能描述**: QC 人员录入质量问题

**输入字段**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| goods_no | string | ✅ | 货品编码（数字） |
| factory_name | string | ✅ | 工厂名称 |
| batch_no | string | ✅ | 波次号（字母 + 数字） |
| issue_type | string | ✅ | 问题类型（多选） |
| issue_desc | string | ✅ | 问题描述 |
| solution_type | string | ✅ | 解决方式 |
| compensation_amount | number | ❌ | 补偿金额（默认 0） |
| product_image | string | ❌ | 商品图 URL |
| issue_images | array | ❌ | 问题图 URL 数组 |
| qc_user_id | int | 自动 | 提交人 ID（自动填充） |
| qc_username | string | 自动 | 提交人用户名（自动填充） |

**业务规则**:
1. 输入波次号自动填充工厂和货品编码
2. 输入货品编码自动填充波次号和工厂
3. 自动记录当前登录用户为提交人
4. issue_no 唯一性保证：时间戳 + UUID + 用户 ID

### 2.3 批次查询

**功能描述**: 根据波次号或货品编码查询关联信息

**接口**:
- `GET /api/batches/{batch_no}` - 根据波次号查询
- `GET /api/batches/search?goods_no=xxx` - 根据货品编码查询

**返回数据**:
```json
{
  "batch_no": "F21140",
  "factory_name": "东遇",
  "goods_no": "23181802104"
}
```

### 2.4 图片上传

**功能描述**: 上传图片到服务器

**接口**: `POST /uploads`

**要求**:
- 支持格式：jpg, png, jpeg
- 自动压缩：最大 1920x1920
- 存储位置：`/app/uploads/`
- 访问 URL：`/uploads/{filename}`

### 2.5 数据查询

**功能描述**: 查询问题列表

**接口**: `GET /api/issues`

**参数**:
- `page` - 页码（默认 1）
- `page_size` - 每页数量（默认 20）
- `factory` - 工厂筛选
- `issue_type` - 问题类型筛选

## 4. 数据库设计

### 4.1 quality_issues 表

```sql
CREATE TABLE quality_issues (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    issue_no VARCHAR(32) UNIQUE NOT NULL,      -- 问题编号（唯一）
    goods_no VARCHAR(32),                       -- 货品编码
    factory_name VARCHAR(64),                   -- 工厂名称
    batch_no VARCHAR(32),                       -- 波次号
    issue_type VARCHAR(255),                    -- 问题类型
    issue_desc TEXT,                            -- 问题描述
    solution_type VARCHAR(32),                  -- 解决方式
    compensation_amount DECIMAL(10,2) DEFAULT 0, -- 补偿金额
    status VARCHAR(16) DEFAULT 'pending',       -- 状态
    qc_user_id INT,                             -- 提交人 ID
    qc_username VARCHAR(32),                    -- 提交人用户名
    product_image VARCHAR(255),                 -- 商品图 URL
    issue_images JSON,                          -- 问题图 URL 数组
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    imported_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_factory (factory_name),
    INDEX idx_status (status),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**关键字段说明**:
- `issue_no`: 唯一标识，生成规则 `Q{YYYYMMDDHHMMSS}{UUID6 位}{用户 ID}`
- `qc_user_id`: 自动从 JWT Token 获取
- `qc_username`: 自动从 JWT Token 获取

### 4.2 product_batches 表

```sql
CREATE TABLE product_batches (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    batch_no VARCHAR(32) UNIQUE,      -- 波次号（唯一）
    factory_name VARCHAR(64),         -- 工厂名称
    goods_no VARCHAR(32)              -- 货品编码
    
    INDEX idx_batch (batch_no),
    INDEX idx_factory (factory_name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**数据量**: 12,696 条

### 4.3 qc_users 表

```sql
CREATE TABLE qc_users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(32) UNIQUE NOT NULL,
    password_hash VARCHAR(128) NOT NULL,
    real_name VARCHAR(32),
    department VARCHAR(32),
    role VARCHAR(16) DEFAULT 'qc',
    status TINYINT DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**默认账号**:
- admin / admin123 (管理员)
- qc_manager / admin123 (经理)
- qc_user / admin123 (质检员)

- Token 存储到 localStorage

### 6.2 问题录入页

**文件**: `mobile/issue-entry.html`

**功能**:
1. 波次号输入 → 自动查询工厂和货品编码
2. 货品编码输入 → 自动查询波次号和工厂
3. 问题类型多选（16 种）
4. 解决方式单选（5 种）
5. 图片上传（最多 8 张）
6. 自动获取当前用户信息
7. 草稿保存功能

**自动填充逻辑**:
```javascript
// 波次号变化时
async function searchBatchByBatchNo() {
    const batchNo = document.getElementById('batchNo').value;
    const response = await fetch(`/api/batches/${batchNo}`);
    const data = await response.json();
    document.getElementById('factory').value = data.factory_name;
    document.getElementById('goodsNo').value = data.goods_no;
}

// 货品编码变化时
async function searchBatchByGoodsNo() {
    const goodsNo = document.getElementById('goodsNo').value;
    const response = await fetch(`/api/batches/search?goods_no=${goodsNo}`);
    const data = await response.json();
    if (data.length > 0) {
        document.getElementById('batchNo').value = data[0].batch_no;
        document.getElementById('factory').value = data[0].factory_name;
    }
}
```
