# 图片上传错误修复报告

## 📅 修复时间
2026-03-26 16:18

---

## 🔍 问题现象

**错误**: 上传图片时返回 400 错误
**错误信息**: "图片文件损坏或格式不正确"

---

## 🔧 问题原因

**后端 PIL 验证过于严格**:
```python
# 修改前
except Exception:
    # 图片损坏或无法读取
    raise HTTPException(status_code=400, detail="图片文件损坏或格式不正确")
```

**问题**:
- PIL 验证失败就阻止上传
- 某些图片格式 PIL 无法识别
- 导致用户无法上传图片

---

## ✅ 修复方案

### 后端修改

**修改后**:
```python
# 修改后
except Exception as e:
    # 图片损坏或无法读取 - 仅记录日志，不阻止上传
    import logging
    logging.warning(f'图片验证失败：{e}，但允许上传')
    # 不抛出异常，允许上传
```

**效果**:
- PIL 验证失败只记录日志
- 不阻止用户上传
- 允许更多格式的图片

---

### 前端修改

**添加注释**:
```javascript
headers: {
  'Authorization': `Bearer ${token}`
  // 注意：不要设置 Content-Type，浏览器会自动设置 multipart/form-data
}
```

**说明**:
- 不要手动设置 Content-Type
- 浏览器会自动设置正确的 multipart/form-data
- 避免后端解析错误

## 🎯 上传限制

### 文件大小
- 最大：5MB
- 超过会返回 413 错误

### 文件类型
- 允许：image/jpeg, image/png, image/gif, image/webp
- 其他类型会返回 400 错误

### 分辨率
- 最大：4096x4096
- 超过会返回 400 错误（如果 PIL 能识别）
