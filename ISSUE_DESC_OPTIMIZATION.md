### 阶段 3: 提交前验证

**验证逻辑**:
```javascript
const issueDesc = document.getElementById('issueDesc').value.trim();

if (!issueDesc) {
  alert('请填写问题描述');
  return;
}

if (issueDesc.length < 10) {
  alert('问题描述至少 10 个字，当前 ' + issueDesc.length + ' 个字');
  return;
}

if (issueDesc.length > 500) {
  alert('问题描述最多 500 个字，当前 ' + issueDesc.length + ' 个字');
  return;
}
```

**错误提示**:
| 场景 | 提示信息 |
|------|----------|
| 未填写 | "请填写问题描述" |
| 少于 10 字 | "问题描述至少 10 个字，当前 X 个字" |
| 超过 500 字 | "问题描述最多 500 个字，当前 X 个字" |

**效果**:
- ✅ 明确错误原因
- ✅ 告知当前字数
- ✅ 指导用户修改

## 📋 完整验证流程

### 正常流程
```
1. 用户看到提示："建议 10-500 字"
2. 用户开始输入
3. 实时显示字数："已输入 50 字"（蓝色）
4. 用户继续输入到 450 字
5. 字数变橙色："已输入 450 字"（橙色）
6. 用户停止输入
7. 点击提交
8. 验证通过 ✅
9. 提交成功
```

### 错误流程 1 - 字数太少
```
1. 用户输入 5 个字
2. 字数显示红色："已输入 5 字"（红色）
3. 用户点击提交
4. 验证失败
5. 提示："问题描述至少 10 个字，当前 5 个字"
6. 用户继续输入到 50 字
7. 字数显示蓝色："已输入 50 字"（蓝色）
8. 点击提交
9. 验证通过 ✅
10. 提交成功
```

### 错误流程 2 - 字数超限
```
1. 用户输入到 500 字
2. 字数显示红色："已输入 500 字"（红色）
3. 显示警告："⚠️ 已达字数上限"
4. 无法继续输入
5. 用户点击提交
6. 验证通过 ✅（刚好 500 字）
7. 提交成功
```
